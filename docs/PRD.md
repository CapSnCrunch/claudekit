## 1. Executive Summary

ClaudeKit is an open-source desktop application that supercharges how developers and power users interact with Claude Code. It consolidates the best ideas from the Claude Code ecosystem — session history browsing, real-time prompt quality feedback, AI-powered prompt rewriting, token/cost tracking, and raw session introspection — into a single, polished, installable app with pre-built GitHub releases.

Users authenticate with their own Anthropic API key (or OAuth token). All AI-powered analysis runs against their own account. No data is sent to any third-party server. Everything runs locally by default.

**Target users:** Developers and power users already using Claude Code (the CLI), who want more visibility into their sessions and want to get better results from Claude over time.

---

## 2. Problem Statement

Claude Code stores rich session data locally, but it's scattered across raw JSONL files that are impossible to browse meaningfully. Several open-source tools have emerged to address individual parts of this problem:

| Tool | What it solves | What it doesn't do |
|------|---------------|-------------------|
| claude-code-history-viewer | Browse sessions visually | No AI analysis, no insights |
| Vibe-Log | Real-time prompt scoring | CLI-only, no unified UI |
| claude-code-prompt-optimizer | Auto-rewrites weak prompts | No history, no context |
| claude-trace | Raw HTTP introspection | Developer tool, not end-user |

None of these are integrated. Installing and configuring them together requires significant manual effort. There is no single installable app that unifies them.

**ClaudeKit is that app.**

---

## 3. Goals & Non-Goals

### Goals

- Provide a polished, installable desktop app (macOS, Windows, Linux)
- Unify session browsing, prompt quality insights, prompt rewriting, token tracking, and raw inspection
- Support both Anthropic API key auth and OAuth token auth (user provides their own credentials)
- Be fully open source with pre-built binaries released on GitHub
- Be local-first — no cloud dependency, no telemetry by default
- Generate actionable insights from session history that help users improve over time

### Non-Goals

- Not a Claude Code replacement — it wraps and enhances Claude Code, doesn't replicate it
- Not a cloud SaaS product (no server-side accounts, billing, or databases in v1)
- Not a general-purpose AI assistant — purpose-built for Claude Code session analysis
- Not a Windows-only or macOS-only tool — cross-platform from day one
- No support for other AI providers in v1

## 4. Users & Use Cases

### Primary User

A developer who uses Claude Code daily (or heavily), works on software projects (but also writing/research), and wants to:

- Understand their patterns and inefficiencies over time
- Get better outputs from Claude by improving how they prompt
- Have a beautiful, fast interface to find old sessions and conversations
- Monitor token usage and avoid surprises on their API bill

### Key Use Cases

**UC-1: Browse session history**
User opens ClaudeKit, sees all their Claude Code projects and sessions in a sidebar, clicks into any session, reads the full conversation with syntax-highlighted code blocks and tool results rendered properly.

**UC-2: Understand prompt quality over time**
User views an "Insights" dashboard that shows their prompt quality score trends, recurring friction points, and actionable suggestions — generated by analyzing their recent sessions.

**UC-3: Get real-time feedback as they type**
When connected to Claude Code (via hook), ClaudeKit shows a prompt quality score in a small overlay or status widget as the user types each prompt, with a one-line suggestion.

**UC-4: Auto-optimize a prompt on demand**
User selects a session's prompt, clicks "Optimize," and ClaudeKit sends it through Claude Opus with extended thinking to produce a detailed, structured version of the prompt.

**UC-5: Monitor token usage and costs**
User sees a dashboard of token usage per project, per day, per model (Haiku vs Sonnet vs Opus), with estimated costs calculated from Anthropic's pricing.

**UC-6: Inspect raw session internals**
Advanced user opens the "Inspector" tab for any session and sees the full HTTP request/response pairs — system prompts, tool definitions, Haiku sub-agent calls — not just the user-visible conversation.

## 5. Product Architecture Overview

### 5.1 Framework

**Tauri v2** — Rust backend + React/TypeScript frontend.

- Lighter and faster than Electron (no bundled Chromium)
- Native system tray, file system access, and OS notifications
- Produces small, signed binaries for macOS, Windows, Linux
- Same tech stack as claude-code-history-viewer (proven viable)

### 5.2 High-Level Architecture

```
┌─────────────────────────────────────────────────────┐
│                  ClaudeKit Desktop App               │
│                                                      │
│  ┌─────────────┐  ┌──────────────┐  ┌────────────┐  │
│  │  React UI   │  │  Tauri       │  │  Rust Core │  │
│  │  (frontend) │◄─┤  Commands &  ├─►│  (backend) │  │
│  │             │  │  Events      │  │            │  │
│  └─────────────┘  └──────────────┘  └─────┬──────┘  │
│                                           │         │
└───────────────────────────────────────────┼─────────┘
                                            │
              ┌─────────────────────────────┼──────────────┐
              │                             │              │
      ┌───────▼─────────┐         ┌─────────▼──────────┐   │
      │  ~/.claude/     │         │  Anthropic API     │   │
      │  JSONL files    │         │  (user's own key)  │   │
      │  Session data   │         │                    │   │
      └─────────────────┘         └────────────────────┘   │
              │                                            │
      ┌───────▼─────────┐         ┌──────────────────────┐ │
      │  Claude Code    │         │  ClaudeKit Hook       │ │
      │  Hook System    │◄────────┤  (UserPromptSubmit)   │ │
      │                 │         │                       │ │
      └─────────────────┘         └───────────────────────┘ │
```

### 5.3 Data Flow

1. **Session data:** Rust backend watches `~/.claude/projects/` for file changes, parses JSONL, indexes into a local SQLite database for fast queries.
2. **AI analysis:** React UI triggers Tauri commands → Rust makes HTTP calls to Anthropic API using user's key → results stored in SQLite and displayed in UI.
3. **Real-time hook:** A separate Node.js/shell script (installed separately by user) intercepts Claude Code prompts → writes to a local Unix socket or file → ClaudeKit watches and displays feedback.
4. **Raw inspection:** Rust reads the same JSONL files but exposes the full message structure including system prompts and tool calls.

### 5.4 Local Storage

All app data stored in the OS app data directory:

- `~/.claudekit/` (macOS/Linux) or `%APPDATA%\claudekit\` (Windows)
- `db.sqlite` — indexed sessions, analysis results, cost tracking
- `config.json` — API key (encrypted at rest), preferences
- `analysis-cache/` — cached HTML reports

## 6. Feature Modules

The app is organized into five feature modules, each corresponding to a development phase.

### Module 1 — Session Browser (Foundation)

*Source inspiration: claude-code-history-viewer*

Browse all Claude Code projects and sessions. Render conversations with syntax highlighting, diff views, and tool result formatting. Full-text search across all sessions. Activity heatmap. This is the foundation all other modules build on.

### Module 2 — Insights & Session Analysis

*Source inspiration: Vibe-Log batch analysis mode*

On-demand AI analysis of recent sessions. Generates a structured HTML report covering: prompt quality trends, what's working, friction patterns, strategic suggestions for improving workflow. Uses Claude via the user's API key. Report is saved locally and viewable in-app.

### Module 3 — Real-Time Prompt Feedback

*Source inspiration: Vibe-Log UserPromptSubmit hook*

Intercepts prompts as they're submitted to Claude Code (via a hook), scores them, and displays a score + one-line tip in a ClaudeKit overlay or status bar widget. Does not block the prompt — purely additive feedback.

### Module 4 — Prompt Optimizer

*Source inspiration: claude-code-prompt-optimizer*

User selects any prompt (from session history or types a new one), clicks "Optimize," and ClaudeKit rewrites it using Claude Opus with extended thinking into a fully structured, detailed prompt. Shows before/after diff. User can copy the optimized version.

### Module 5 — Token Tracker & Raw Inspector

*Source inspiration: claude-trace + History Viewer analytics*

**Token Tracker:** Per-project, per-day, per-model token usage dashboard. Estimated cost display based on current Anthropic pricing. Usage trend charts.

**Raw Inspector:** Full HTTP-level view of any session — system prompts, tool definitions, all model calls including Haiku sub-agent calls. Intended for power users who want to understand what Claude Code is actually doing.

## 7. Development Phases

Each phase has its own PRD and TDD, which are derived from this master document.

### Phase 1 — Foundation: Session Browser

**Deliverable:** Working Tauri app with session browsing, search, and basic analytics.
**Duration:** ~2–3 weeks
**Dependencies:** None (reads local JSONL files only, no API key required)
**Success criteria:**
- App installs and runs on macOS, Windows, Linux
- All sessions from `~/.claude` are visible and browsable
- Full-text search works across all sessions
- Syntax-highlighted code blocks render correctly
- Activity heatmap visible on dashboard

### Phase 2 — Insights Engine

**Deliverable:** AI-powered session analysis with HTML report generation.
**Duration:** ~2 weeks
**Dependencies:** Phase 1 complete; Anthropic API key required
**Success criteria:**
- User can trigger analysis of last N sessions
- Report generates within 60 seconds for a typical user
- Report covers: prompt quality score, top friction points, top wins, actionable suggestions
- Report is saved and viewable in-app, also exportable as HTML file

### Phase 3 — Real-Time Prompt Feedback

**Deliverable:** Hook integration + live prompt scoring overlay.
**Duration:** ~2 weeks
**Dependencies:** Phase 1 complete; Anthropic API key required
**Success criteria:**
- Hook installs with a single command from within ClaudeKit
- Prompt score appears within 3 seconds of submitting a prompt
- Score + tip displayed in ClaudeKit status widget without interrupting Claude Code
- Hook can be toggled on/off from within ClaudeKit

### Phase 4 — Prompt Optimizer

**Deliverable:** In-app prompt rewriting using Claude Opus + extended thinking.
**Duration:** ~1–2 weeks
**Dependencies:** Phase 1 complete; Anthropic API key required (Opus access)
**Success criteria:**
- User can select any prompt from session history and click "Optimize"
- Optimized prompt shown in before/after diff view
- Copy-to-clipboard works
- Optimization completes within 10 seconds on average

### Phase 5 — Token Tracker & Raw Inspector

**Deliverable:** Full token usage dashboard + raw HTTP session inspector.
**Duration:** ~2 weeks
**Dependencies:** Phase 1 complete
**Success criteria:**
- Token usage visible per project, per day, per model
- Cost estimate displayed using latest Anthropic pricing (fetched periodically)
- Raw inspector shows complete message structure for any session
- System prompt, tool definitions, and sub-agent calls all visible

## 8. Technical Constraints & Decisions

### 8.1 Language & Runtime

- **Rust (backend):** File watching, JSONL parsing, SQLite, HTTP calls to Anthropic API, OS integration
- **TypeScript + React (frontend):** All UI, charts, report rendering
- **Node.js (hook script):** UserPromptSubmit hook — separate from the app, installed by ClaudeKit

### 8.2 Key Dependencies

| Dependency | Purpose |
|-----------|---------|
| tauri v2 | Desktop app framework |
| rusqlite | SQLite in Rust |
| notify (Rust) | File system watching |
| serde_json | JSONL parsing |
| reqwest | HTTP client for Anthropic API |
| react + vite | Frontend |
| recharts | Charts and graphs |
| shadcn/ui + tailwind | UI components |
| @anthropic-ai/sdk | Hook script (Node.js) |
| better-sqlite3 | Hook script local writes |

### 8.3 Auth & Security

- API key stored encrypted using OS keychain (macOS Keychain, Windows Credential Manager, Linux libsecret via `keyring` crate)
- API key never written to plain config files
- No telemetry, no analytics, no external server calls except to `api.anthropic.com`
- All session data stays on disk, never uploaded

### 8.4 Platform-Specific Considerations

| Platform | Notes |
|---------|-------|
| macOS | Universal binary (Intel + Apple Silicon). Code signing required for Gatekeeper. |
| Windows | NSIS installer. Windows Defender may flag unsigned binaries — signing certificate needed for polished release. |
| Linux | AppImage + .deb + .rpm. No signing required but AppImage is most portable. |

### 8.5 Hook Architecture

The real-time prompt feedback hook (Phase 3) runs as a separate process outside the Tauri app — it's a Node.js script registered with Claude Code's hook system. Communication between the hook and the app happens via:

- **Option A (recommended):** Named pipe / Unix domain socket — hook writes score JSON, app reads it
- **Option B (fallback):** Shared file in `~/.claudekit/hook-output.json` with file watcher

The app manages hook installation/uninstallation, but the hook process itself is independent.

## 9. UX Principles

1. **Non-intrusive by default.** Real-time feedback should not slow down or interrupt Claude Code usage.
2. **Progressive disclosure.** Basic features (session browser) work without an API key. AI features prompt for key only when needed.
3. **Local-first.** Everything works offline except AI analysis features.
4. **Fast.** Session browser should feel instant even with thousands of sessions. SQLite indexing is key.
5. **Transparent.** Users can always see what data the app is reading and what API calls it's making.

## 13. Open Questions

1. **Hook communication protocol:** Named pipe vs. shared file — need to validate named pipe support on Windows.
2. **Pricing data freshness:** How often should the app refresh Anthropic's pricing table for cost estimates? Monthly seems reasonable.
3. **Report LLM choice:** Should the Insights Engine use Haiku (cheap, fast) or Sonnet (better quality) by default? Consider making it user-configurable.
4. **claude-trace integration depth:** Full HTTP interception requires patching Claude Code's binary. Is this sustainable as Claude Code updates? Consider limiting raw inspector to what's available in JSONL files for v1.
5. **Windows hook support:** Claude Code hooks use bash scripts. Windows support requires WSL or a PowerShell equivalent. May need to scope Windows hook support to WSL users only for Phase 3.
